# PRÁCTICA 3 — Requests y Responses manuales

## Objetivo

Entender cómo funcionan las vistas basadas en funciones con DRF.

---

## Paso 1 — Crear vista con @api_view

En `api/views.py`:

```python
from rest_framework.decorators import api_view
from rest_framework.response import Response
from rest_framework import status
from .models import Producto
from .serializers import ProductoSerializer

@api_view(['GET', 'POST'])
def producto_list(request):
    if request.method == 'GET':
        productos = Producto.objects.all()
        serializer = ProductoSerializer(productos, many=True)
        return Response(serializer.data)

    if request.method == 'POST':
        serializer = ProductoSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)
```

---

## Paso 2 — Registrar ruta manual

En `urls.py`:

```python
from api.views import producto_list

urlpatterns += [
    path('api/productos-manual/', producto_list),
]
```
